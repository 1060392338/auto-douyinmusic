## 一、项目概览

自动化从AI音乐创作到导出的完整流程。基于5Agent架构（DeepSeek双模型），通过浏览器自动化（DrissionPage）操作抖音音乐开放平台。

**已实现的核心能力：**
- ✅ AI作曲 → 编辑器 → 导出到资产页（可靠批量导出，11/12首已完成）
- 👤 全自动发布全曲（已转为人工操作，用户浏览器手动完成）

## 四、踩过的坑（完整记录）

### 坑1：`--headless=new` 与 DrissionPage WebSocket 404

**现象**：`ChromiumPage(addr_or_opts='127.0.0.1:9223')` 报 "WebSocket 404 Not Found"
**原因**：Chrome 147+ 的 `--headless=new` 模式下，DrissionPage 4.1.1.2 的 WebSocket 连接方式不兼容
**解法**：改用窗口隐藏模式 `--window-position=-32000,-32000`，等效无头但 WebSocket 完全正常

### 坑2：DrissionPage 自动启动 Chrome 到临时目录（静默丢 Session）

**现象**：之前登录的 session 全部丢失，Chrome 打开的是空白新实例
**原因**：当 9223 端口无 Chrome 时，`ChromiumPage(addr_or_opts='127.0.0.1:9223')` **静默**启动 Chrome 到临时目录 `/var/folders/.../T/DrissionPage/userData/9223`
**解法**：先手动启动 Chrome（指定正确 `--user-data-dir`），等 2-3 秒后再用 ChromiumPage 连接
**检测**：`ps aux | grep chrome.*9223 | grep -o 'user-data-dir=[^ ]*'`

### 坑3：点击导出按钮不触发弹窗（点了内层 span 而非 button）

**现象**：JS 点了"导出"文字但弹窗不出现
**原因**：Semi UI 的按钮结构是 `<button> → <span class="button-content"> → <span class="button-content-right">导出</span></span>`，React 的事件绑定在 `<button>` 上，点内层 span 无效
**解法**：必须点 `button.semi-button-secondary` 本身，用 `mousedown+mouseup+click` 全事件链
```javascript
document.querySelectorAll('button.semi-button-secondary').forEach(b => {
    if (b.textContent.trim() === '导出') {
        ['mousedown','mouseup','click'].forEach(type => {
            b.dispatchEvent(new MouseEvent(type, {bubbles:true, cancelable:true, view:window, button:0}));
        });
    }
});
```

### 坑4：歌名变成 `[object HTMLInputElement]`

**现象**：导出对话框输入值修改后，资产页显示歌名为 `[object HTMLInputElement]`
**原因**：在 disabled 状态的 input 上用 nativeSetter 改值前，没有先 `disabled = false`。disabled 状态下的 input 行为异常
**解法**：三步走 —— ① `inp.disabled = false` ② nativeSetter 改值 ③ dispatchEvent input+change

### 坑5：脚本被杀后 Chrome 残留（暂停不彻底）

**现象**：用户说"暂停"后 Chrome 还在跑，继续在作曲页/编辑器操作
**原因**：Hermes 的 `process(kill)` 杀掉 Hermes 后台进程，但 Python 子进程和 Chrome 进程独立存活
**解法**：在脚本开头注册信号处理器，收到 SIGTERM 时主动杀 Chrome：
```python
def _cleanup(signum, frame):
    os.system("lsof -ti :9223 | xargs kill -9 2>/dev/null")
    os._exit(0)
signal.signal(signal.SIGTERM, _cleanup)
signal.signal(signal.SIGINT, _cleanup)
```

### 坑6：SPA 页面需要等待歌曲加载完成才点导出

**现象**：进入编辑器后立即点导出，弹窗不出现或按钮点击无响应
**原因**：编辑器需要时间加载歌曲的音频/波形数据，期间导出按钮不可用
**解法**：进入 playground 后等 5-8 秒再点击导出

### 坑7：`AlertExistsError` 阻塞所有 JS 操作

**现象**：`page.run_js()` 报 `AlertExistsError: 存在未处理的提示框`
**原因**：beforeunload 弹窗或其他 alert 对话框未处理，浏览器阻止 JS 执行
**解法**：每次 run_js 前先 `try: page.handle_alert(accept=True); except: pass`

### 坑8：AI 编辑 / 去 AI 编辑器按钮不是 `.semi-button`

**现象**：用 `button.semi-button` 选择器找不到 AI 编辑按钮
**原因**：AI 编辑的类名是 `aiEditButton-BxDygS`，去 AI 编辑器的类名是 `primaryBtn-gl7Q_G`，都不是 Semi UI 的按钮类
**解法**：用独立选择器：`button[class*="aiEditButton"]` 和 `button[class*="primaryBtn"]`

### 坑9：导出弹窗歌名 input 是 disabled 的

**现象**：能看到导出对话框中有歌名输入框但无法修改
**原因**：Semi Design 的 Input 组件在导出对话框中是 readonly/disabled 状态
**解法**：先 `disabled = false; removeAttribute('disabled')` 再改值

### 坑10：批量导出时 for 循环内的 Python 进程独立存活

**现象**：Shell for 循环被杀后，内部的 `python3 export_v5.py xxx` 还在跑
**原因**：`for song in ...; do python3 ...; done` — shell 被杀时 Python 子进程成为孤儿进程
**解法**：单首导出（不批量），每次只跑 `export_v5.py <一个歌名>`

## 五、注意事项

1. **端口隔离**：头条用 9222，音乐用 9223，两项目同时运行不冲突
2. **单首导出**：不要用 shell for 循环批量，每次只导出 1 首
3. **等 Chrome 就绪**：Chrome 启动后等 2-3 秒再连接
4. **信号处理**：export_v5.py 已有 SIGTERM 清理，用 `process(kill)` 即可完全停止
5. **session TTL**：抖音 session 约 5-10 分钟过期，需要重新登录
6. **`$HOME` 变量**：在 `terminal(background=true)` 中不会展开，须用绝对路径
7. **发布全曲**：由用户人工操作，在浏览器中手动完成选曲→签约→发布
